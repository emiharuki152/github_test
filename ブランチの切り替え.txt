git checkout -b feature/test-branch とは？
1. git checkout の意味
「ブランチを切り替える」コマンドです。

たとえば git checkout main は「mainブランチに移動して、その状態に作業ツリーを切り替える」という意味。

2. -b オプションの意味
-b は「新しいブランチを作る」＋「その新しいブランチに切り替える」

つまりこの1回のコマンドで「ブランチ作成」と「切り替え」の両方ができます。

3. feature/test-branch は新しいブランチ名
ブランチ名は自由に付けられますが、

「feature/〜」のように用途ごとに名前を付けるのが一般的（例: 機能追加ならfeature、バグ修正ならbugfixなど）

実際の流れと使いどころ
例えば：

git checkout -b feature/add-login
「feature/add-login」という新しいブランチを作って、そこに切り替わる

以降はそのブランチで作業を進められる（ファイル編集、コミットなど）

どうしてブランチを使うの？
main（メイン）ブランチは安定した状態を保つのが理想

新しい機能や修正は別ブランチで作業し、問題なければmainにマージする

これで複数人でも同時に違う作業ができるし、mainを壊すリスクを減らせる

まとめ
コマンド	説明
git branch	今あるブランチ一覧を見る
git checkout ブランチ名	既存のブランチに切り替える
git checkout -b 新しいブランチ名	新しいブランチを作って切り替える
